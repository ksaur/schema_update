Module json_patch_creator

source code

This is the main file for generating the update code.

Two modes:

  1. generate a template to fill out from two (old and new) json test data files
  2. generate the python code to perform the update using the template file

Requires Python 2.7.

generate_add_key(keyglob, usercode, outfile, prefix)

source code 

Generate code to add databasekeys. Handles range nastiness. Ex: ea_n[1-2]@n[1-3,5] becomes ['ea_n1@n1', 'ea_n1@n2', ...'ea_n2@n5']

Parameters:
  • keyglob - the limited keyglob of keys to add. (ranges but no wildcards)
  • usercode (string) - the code to initialize the new key
  • outfile - the file to write the generated function to
  • prefix - a string to ensure function name uniqueness
Returns:
a dictionary of ( key blob command -> functions to call)

Takes as input the name of a DSL file in the format of: for * { .....(Rules beginning with INIT, UPD, REN, DEL).....}

To generate a template to fill out (containing the added/deleted fields between 2 json files):

>>> python json_patch_creator.py --t ../tests/data/example_json/sample1.json ../tests/data/example_json/sample2.json

To generate the update code from the template:

>>> python json_patch_creator.py --d ../tests/data/example_json/sample_init
