 25  """ 
 26      The following modifications were made to address key versioning: 
 27      * All keys are prepended with a version number and pipe.   
 28          ex:  key0 at version v0 becomes "v0|key0". 
 29   
 30      To make redis operation work, this class: 
 31      * Tries prepending all possible version strings for: 
 32          delete, get 
 33      * Prepends only the most recent version string for: 
 34          set 
 35      * Prepend wildcard (*|) to keyglob for: 
 36          keys 
 37     
 38      Also, 'flushall' preserves the stored list of versions 
 39   
 40      Additional bookkeeping for lazy update: 
 41      * A redis list called "UPDATE_VERSIONS_NS", which is a list of version name strings 
 42           for each namespace NS.  (There will be a list of versions for each namespaces) 
 43           Ex: "UPDATE_VERSIONS_NS" -> ["V1"] 
 44   
 45      * A redis hash of "UPDATE_DSL_STRINGS", which stores a hash mapping version name strings 
 46           concatinated with namespaces to DSL strings (for generating to modules later by  
 47           other connected clients into the self.upd_dict  
 48           as clients connect)  One "UPDATE_DSL_STRINGS" hash total. 
 49           Ex:  "UPDATE_DSL_STRINGS" -> { "(V1, V2, NS)" : "for ns:... v0->v1",  
 50                "(V1, V2, NS2)" : "add ns2....v0"} 
 51      * Update functions are stored in a dictionary named self.upd_dict 
 52        This stores "version string+ -> (version module, dict of (keyglob->module function strings)) 
 53           Ex: TODO  
 54    
 55      
 56      The initial (preupdate) version is named "INITIAL_V0"  
 57      (global string "initial_version" for now).  Subsequent 
 58      versions will be named when the user passes in the set of corresponding 
 59      update functions.
